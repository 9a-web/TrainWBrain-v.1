# PRD — TrainWithBrain (память агента)

> Канонические документы проекта: **`/app/twb_plan.md`** (ТЗ + архитектура + дорожная карта) и **`/app/AI_CONTEXT.md`** (техническая карта кода). Этот файл — краткая память для агентов.

## Проблема / продукт
Приложение для трекинга силовых тренировок (пауэрлифтинг) с двумя ролями — **спортсмен** и **тренер**. Работает как сайт, Telegram Mini App и PWA. Тренер ведёт подопечных, готовит планы, в реальном времени видит и подтверждает тренировку; спортсмен запускает тренировку и по-подходно отмечает выполнение.

## Стек
React 19 (CRACO) + FastAPI + MongoDB (Motor). Backend — монолит `server.py` (~3400 строк) + `models.py`/`auth.py`/`realtime.py`/`seed.py`. Frontend — `src/` (App, api, components, pages, context, hooks).

## Реализовано (актуально на Июнь 2026)
- **Auth (обязательный):** email/пароль (bcrypt), Telegram initData (HMAC), Google (прямой OAuth + Emergent). Сессии `user_sessions`, Bearer `twb_token`. **IDOR закрыт** на `/sessions/*` и `/plans/*` (401/403).
- **P1** Программы (29 упражнений, 4 шаблона), снимок плана, реальный прогресс.
- **P2** Жизненный цикл тренировки, **по-подходный лог + таймер отдыха + настройки**, тоннаж/%1ПМ/streak.
- **P2.1** Пропуски/переносы (`plan_day_marks`), adherence.
- **P3** Режим тренера: invite/link, подопечные, редактор плана, draft/published, недели, тренировочные дни, **подтверждение тренером**, **старт тренировки тренером**.
- **P4** Real-time WebSocket (`/api/ws`), co-scribe (`filled_by`/`coach_confirmed`), `useRealtime`.
- **P7** Подробная статистика с графиками (recharts), coach-gated эндпоинты.
- **CP** Кроссплатформенность: website + Telegram + PWA.

## Статистика: багфиксы B1–B11 + редизайн (Июнь 2026, эта сессия)
- **B1 IDOR закрыт на `/stats/*`:** все 4 роута `/stats/{tg}*` требуют `Depends(get_current_user)` + `_assert_can_read_stats` (владелец или активный тренер); coach-gated `/coach/{coach_id}/clients/{ath}/stats|exercise-progress` проверяют caller==coach_id.
- **B2/B9:** единый анкер даты `_session_day(s)` (date → finished_at) во всех расчётах; сессии без даты исключаются из range-фильтра.
- **B3:** `_est_1rm` — формула Эпли по фактическим весу/повторам (fallback: вес ÷ план%). 1ПМ/мышцы в detailed считаются свежо из упражнений (не из замороженного снимка).
- **B4:** exercise-progress без `plan_id` = «всё время» (все планы, группировка по датам, `scope: all`); с `plan_id` — по микроциклам.
- **B5:** в списке упражнений только те, где есть рабочий вес.
- **B6:** `avg_per_week` уважает скоуп (на микроцикл при plan_id, иначе по календарю).
- **B7:** `completion_pct`/`volume_pct` ограничены 100%.
- **B8:** подсобные упражнения учитываются в распределении по группам (≥1 подход).
- **B10:** подпись «по микроциклам» только при реальном плане (frontend).
- **B11:** streak `total_workouts` = число сессий (как в Stats) + новое поле `active_days`; в календаре `count` на день.
- **Редизайн `/stats`:** bento-hero (тоннаж + спарклайн, кольцо выполнения, серия, мини-KPI), 1ПМ-карточки факт(Эпли)/план с прогресс-баром, карточка «Динамика» (табы тоннаж/частота, ReferenceLine среднего), «Прогресс упражнения» с метрик-чипами (Вес/1ПМ/Тоннаж, AreaChart с градиентом), донат групп мышц с %-легендой, skeleton-загрузка, ambient-glow, GG Zaglav для больших чисел.
- **Редизайн `/streak`:** hero с гигантским числом (GG Zaglav, градиент, glow), подстаты-карточки, прогресс-бар цели недели, хитмап с метками месяцев и интенсивностью (count≥2 = hot), выровненные лейблы дней.
