# PRD — TrainWithBrain (память агента)

> Канонические документы проекта: **`/app/twb_plan.md`** (ТЗ + архитектура + дорожная карта) и **`/app/AI_CONTEXT.md`** (техническая карта кода). Этот файл — краткая память для агентов.

## Проблема / продукт
Приложение для трекинга силовых тренировок (пауэрлифтинг) с двумя ролями — **спортсмен** и **тренер**. Работает как сайт, Telegram Mini App и PWA. Тренер ведёт подопечных, готовит планы, в реальном времени видит и подтверждает тренировку; спортсмен запускает тренировку и по-подходно отмечает выполнение.

## Стек
React 19 (CRACO) + FastAPI + MongoDB (Motor). Backend — монолит `server.py` (~3400 строк) + `models.py`/`auth.py`/`realtime.py`/`seed.py`. Frontend — `src/` (App, api, components, pages, context, hooks).

## Реализовано (актуально на Июнь 2026)
- **Auth (обязательный):** email/пароль (bcrypt), Telegram initData (HMAC), Google (прямой OAuth + Emergent). Сессии `user_sessions`, Bearer `twb_token`. **IDOR закрыт** на `/sessions/*` и `/plans/*` (401/403).
- **P1** Программы (29 упражнений, 4 шаблона), снимок плана, реальный прогресс.
- **P2** Жизненный цикл тренировки, **по-подходный лог + таймер отдыха + настройки**, тоннаж/%1ПМ/streak.
- **P2.1** Пропуски/переносы (`plan_day_marks`), adherence.
- **P3** Режим тренера: invite/link, подопечные, редактор плана, draft/published, недели, тренировочные дни, **подтверждение тренером**, **старт тренировки тренером**.
- **P4** Real-time WebSocket (`/api/ws`), co-scribe (`filled_by`/`coach_confirmed`), `useRealtime`.
- **P7** Подробная статистика с графиками (recharts), coach-gated эндпоинты.
- **CP** Кроссплатформенность: website + Telegram + PWA.
